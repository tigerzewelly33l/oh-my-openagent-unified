# OpenCodeKit

OpenCodeKit is a Node.js + TypeScript CLI and template for bootstrapping and managing OpenCode projects with an opinionated multi-agent workflow.

## What This Template Includes

- Beads-first lifecycle: `/create` -> `/start` -> `/ship`
- Custom agents in `.opencode/agent/` (9 in this template)
- Slash commands in `.opencode/command/` (19 workflows)
- Skills in `.opencode/skill/` (large curated library)
- Template-local tools in `.opencode/tool/` (`context7`, `grepsearch`); memory and observation tooling comes from the template plugin layer, while session browsing and skill-MCP runtime surfaces come from the OMO plugin layer
- Plugins in `.opencode/plugin/` for memory, Stitch, and Copilot authentication support

## Quick Start

```bash
# Scaffold or initialize
npx opencodekit init

# Or in this repository during development
npm install
npm run dev -- --help
```

## Core Slash Command Workflow

Use these inside OpenCode:

1. `/create <description>` - create bead and PRD
2. `/start <bead-id>` - claim bead and prepare branch/workspace
3. `/ship <bead-id>` - implement, verify, review, close

## CLI Command Surface (`ock`)

The packaged CLI commands are:

- `ock init`
- `ock activate [key]`
- `ock license [action]`
- `ock agent <list|create|view|remove>`
- `ock command <list|create|show|delete>`
- `ock config [action]`
- `ock doctor`
- `ock status`
- `ock upgrade`
- `ock patch [action]`
- `ock completion [shell]`
- `ock tui`

OCK remains the bootstrap and management front door only. Runtime execution stays on the OMO CLI; there is intentionally no `ock run` command in this slice.

See `CLI.md` for command usage details.

## Repository Layout

```text
src/                  # CLI implementation
.opencode/            # Agents, commands, skills, tools, plugins
```
